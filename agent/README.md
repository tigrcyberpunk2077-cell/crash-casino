# ИИ-агент сетки Telegram-каналов (каналы-персоны)

Ведёт несколько Telegram-каналов, **каждый от лица своей персоны** (например,
девушка-эксперт по трейдингу или ставкам). Сам придумывает темы, пишет посты
от первого лица в характере персоны, рисует картинку и публикует — **но только
после твоего одобрения**. Черновики приходят в бот с кнопками
✅ Опубликовать / ✏️ Другой вариант / ❌ Отклонить.

Один бот = вся сетка: у каждого канала своя персона и ниша (`/addchannel`,
`/persona`). Начни с одного канала, добавляй ещё по мере надобности.

Подача намеренно **образовательная**: без обещаний гарантированной прибыли,
без «сигналов покупай сейчас» и фейковых пруфов, по-человечески про риск.
Так честнее и безопаснее для канала.

## Как устроено (простой режим)

```
                    ┌─────────────── один процесс ───────────────┐
  расписание  →     │  планировщик → ИИ пишет пост + картинку     │
                    │                                             │
  ты (Telegram) ←───┤  бот: шлёт черновик с кнопками              │
                    │  ↑ нажал «Опубликовать»                     │
  канал        ←────┤  тот же бот постит в канал (он там админ)   │
                    └─────────────────────────────────────────────┘
```

Один бот (от @BotFather) делает всё: и присылает черновики, и сам публикует.
**Не нужны** ни аккаунт, ни телефон, ни коды Telegram. Нужно лишь добавить бота
**админом канала** (с правом «Публикация сообщений»). Для подписчиков пост
выглядит как обычный пост канала.

- **LLM**: `gemini` (бесплатно) или `claude` (платный API). Переключается в `.env`.
- **Картинки**: Pollinations.ai — бесплатно, без ключа.
- Состояние (каналы, черновики) — в SQLite (`agent.db`).

## Настройка (один раз)

1. Зависимости:
   ```bash
   source .venv/bin/activate
   pip install -r requirements.txt
   ```
2. Заполни в `.env` (шаблон — в `.env.example`):
   - `AGENT_BOT_TOKEN` — токен бота от @BotFather
   - `AGENT_ADMIN_ID` — твой user-id (узнать у @userinfobot)
   - `LLM_PROVIDER=gemini` + `GEMINI_API_KEY` (бесплатно: aistudio.google.com/apikey)
   - (необязательно) `AGENT_CTA_LINK` — ссылка для призыва; пусто — без ссылок
3. **Открой бота в Telegram и нажми Start** (иначе он не сможет писать тебе первым).
4. **Добавь бота админом своего канала** с правом «Публикация сообщений».

## Запуск

```bash
python -m agent
```

В боте напиши `/help`. Дальше:

1. Добавь канал с персоной:
   ```
   /addchannel @my_channel | трейдинг для новичков | Алина, 26, тёплый наставник, объясняет просто
   ```
   Формат: `@канал | ниша | персона`. Персона — кто «ведёт» канал: имя, характер,
   манера. Чем живее описание — тем органичнее посты.
2. `/now <id>` — сразу сгенерировать черновик для проверки.
3. Дальше планировщик сам предлагает посты в «лучшее время» (см. ниже).

### Команды бота

| Команда | Что делает |
|---|---|
| `/addchannel @канал \| ниша \| персона` | добавить канал в сетку |
| `/persona <id> <описание>` | изменить персону канала |
| `/channels` | список каналов (ниша, персона, статус) |
| `/interval <id> <минуты>` | частота постов для канала |
| `/pause <id>` / `/resume <id>` | выключить / включить канал |
| `/now <id>` | сгенерировать черновик прямо сейчас |

## Картинки к постам

К каждому посту агент генерирует картинку через **Pollinations.ai** — бесплатно,
без ключа. Пост уходит в канал как фото с подписью. Выключить: `AGENT_IMAGES=false`.

## «Лучшее время» публикации

| Переменная | Смысл |
|---|---|
| `AGENT_UTC_OFFSET` | твой пояс относительно UTC (Москва = `3`) |
| `AGENT_ACTIVE_HOURS` | окно постинга, локальное (`10-23`; можно через полночь: `22-2`) |
| `AGENT_PEAK_HOURS` | пиковые часы (`12,15,19,21`); пусто = любой час окна |
| `AGENT_INTERVAL_MIN` | минимальный промежуток между постами в одном канале |

Пост выходит, когда: сейчас внутри окна **и** это пиковый час **и** прошло не
меньше `AGENT_INTERVAL_MIN` с прошлого поста. ⚠️ Пиковые часы должны попадать
в окно — иначе постить негде (агент предупредит в логе при старте).

## Деплой на Render (рядом с казино)

В `render.yaml` уже добавлен второй сервис `channel-agent` (тот же Docker-образ,
команда `python -m agent`).

1. В дашборде Render впиши секреты сервиса `channel-agent`: `AGENT_BOT_TOKEN`,
   `AGENT_ADMIN_ID`, `GEMINI_API_KEY` (или `ANTHROPIC_API_KEY` при claude).
2. Каналы задай через `AGENT_CHANNELS` (JSON) — на сервере SQLite сбрасывается
   при редеплое, а из этой переменной агент пересоздаёт каналы при старте:
   ```
   AGENT_CHANNELS=[{"ref":"@chan","niche":"трейдинг","persona":"Алина, 26, …"}]
   ```

⚠️ **Про тариф:** агент — фоновый процесс (`worker`), а у Render **нет бесплатного
плана для worker**. Варианты: платный `starter` (≈$7/мес), либо бесплатно — на
своём сервере/VPS, fly.io, railway, либо просто держать запущенным локально.

## Замечания

- В **канале** подписчики не видят, что постит бот — пост идёт от имени канала.
  Держи в настройках канала выключенной «Подпись авторов».
- Очередь одобрения защищает от «улетело не то»: ничего не публикуется без твоего
  нажатия. Не нравится пост — жми «✏️ Другой вариант» или «❌ Отклонить».
- Если отклонил черновик — следующий придёт через стандартный интервал (или жми `/now`).
